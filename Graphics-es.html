Unidad Dos -- Graficos

Esta unidad se centra en trabajar con imágenes que, a diferencia de las fotografías, no fueron creados con una cámara, pero en vez crea directamente en el equipo.  Estos tipos de gráficos (graphics) son útiles para cosas como folletos, portadas de revistas, logotipos de empreza, publicidad, o su tarjeta de Navidad de la familia.

Sin embargo, hay una parte muy importante de los gráficos que este curso sólo tocar un poco...los elementos artísticos.  Para hacer buenos gráficos, usted debe entender lo que las imágenes son más agradables a la vista.  Aunque esta sección se ofrecen algunas indicaciones en el camino, para entender realmente esto toma varios cursos de estudio, años de práctica y un montón de trabajo duro.  En su lugar, este curso se centrará en algunos de los fundamentos de cómo los gráficos de trabajo, y cómo utilizar algunas herramientas para llegar a algunos diseños muy básicos.

Gráficos Vectoriales

A diferencia de las imágenes raster (fotografías, cubierta en la Unidad Uno) gráficos vectoriales no se trata de píxeles dispuestos en una cuadrícula con valores diferentes.  En vez de gráficos vectoriales se almacenan como conjuntos de instrucciones que una computadora puede ejecutar para volver a crear un gráfico.

Un ejemplo simple de estas instrucciones pueden ser:

  1. Crear un círculo con un centro de 3 unidades por encima del centro, y un radio de 2 unidades.

  2. Rellene el círculo de color amarillo.

  3. Crear una línea de 2 unidades por debajo del centro de 3 unidades por encima y por dos unidades a la izquierda del centro.

  4. Crear una línea de 2 unidades por debajo del centro de 3 unidades por encima y por dos unidades a la derecha del centro.

Si hubiera un programa que podría ejecutar este cript muy sencillo, puede llamar a la pantalla el texto siguiente:

Lo bueno de esto es que nunca lo más probable es que mirar realmente en la secuencia de comandos de la computadora crea, al igual que usted no tendrá  que ir a la matriz de píxeles y los canales que se almacenan en una fotografía.  Sin embargo, si usted desea conseguir avanzada, que sería posible (y un poco más fácil de editar una imagen de mapa bits).  En su lugar, vamos a utilizar las herramientas que convierten las cosas que dibujar con el ratón y el teclado en una de estas secuencias de comandos que el equipo puede seguir cuando se quiere volver a producir una imagen para ver en la pantalla (la  pantalla es en realidad un mapa de bits). Entonces, cuando todos se hacen, podemos hacer que se ejecute el secuencia de comandos de nuevo para producir otra trama gráficos que se pueden guardar en un archivo PNG.

También es posible que esta secuencia de comandos para incluir una cierta cantidad de tiempo que debe transcurrir antes de que algo sucede.  Esto permite que para las animaciones, y algunos ejemplos sencillos sería tener un moviemiento palo cifra a través de la imagen o tener un rebote de la pelota.

Una de las beneficios muy importantes de las imágenes vectoriales es que todos los comandos de la imagen son relativos.  Si usted necesita para aumentar el tamaño de la imagen, sólo tiene que cambiar las unidades que todos los comandos se aplican.  En el ejemplo anterior, una unidad equivale a aproximadamete 10 píxeles, pere esto podría ser fácilmente modificado para que un píxel es igual a 20 píxeles o incluso 100 píxeles.  La imagen automáticamente escala hasta el nuevo tamaño sin perder calidad, la imagen se quedará crujiente y no podrás ver los artefactors de píxeles.  Esto es lo contrario de la cantidad de fotos de trabajo, allí si usted fuera a tratar de escala de la foto hasta el doble o diez veces el tamaño original, verá problemas aparecen en la imagen.

Usted puede ver que a medida que hablamos en la sección de fotos, el mapa de bits gráficos no se escala hasta el tamaño grande muy bien.  Sin embargo, el gráfico vectorial sigue buscando lisa y limpia en cualquier tamaño.

In addition to cleanly scaling, vector graphics usually take up less space.  In a raster image, data needs to be stored for every pixel (in three or four channels).  If you have large images, this can be millions of bytes.  Vector graphics on the other hand only store the information necessary to make the shapes and color them in.  For the above example, this is just four commands, each one taking only a few bytes.  There are things you can do to make very complicated vector graphics, but typically they will be much smaller than their raster counterparts.

The other very nice thing about vector graphics is that when you are using graphical programs like Inkscape or Illustrator to edit them, it is very easy to select objects.  For photos we had some real challenges in selecting different objects (like the airplane and the statue's arm).  However, since in vector graphics the commands to create each object are stored, when you click on an object, it is easy for the program to guess which object you are trying to select.  You can then easily modify that object by resizing it, rotating it, or moving it.

File Formats

The primary file format currently in use is SVG or Scalable Vector Graphics (see: http://en.wikipedia.org/wiki/Svg ) which is a type of human readable file called XML (eXtensible Markup Language see: http://en.wikipedia.org/wiki/XML). Although XML and SVG are human-readable, we will not be reading them directly as part of this material. SVG is supported in most vector drawing programs, however it is not yet supported by all web browsers (though this is rapidly changing). If you simply want to use your images on web pages, it is best to convert them to Portable Network Graphics (PNG) which is a lossless raster format.

The other major format for Vector Graphics is the EPS or Encapsulated Post Script. Post Script itself is a file type used for printing (especially industrial printing), and EPS images are built to be easily put inside of a Post Script document. This format has been around a lot longer than SVG, and is supported by many more tools (especially the old ones). Likewise, it is not recommended for use with web browsers.
